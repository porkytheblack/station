The signal

Background jobs shouldn't require babysitting. Signal runs your scheduled tasks with the reliability of infrastructure that's been running longer than anyone's been watching — and asks nothing from you in return.

The rust, the overgrown fence, the bird nests — these aren't decay. They're proof of time passing without failure.

03 — Motion

Motion that
doesn't shout.

Animation should feel like the tower — patient, purposeful, indifferent to being noticed.

Pattern 01
Signal Pulse

Expanding rings from the antenna tip. Used for status indicators, the logo mark, "transmitting" states. Three rings offset by 800ms each. Never speeds up — it's always the same cadence.

replay
duration: 2.4s
offset: 800ms
easing: ease-out
count: infinite
Pattern 02
Status Blink

A single dot that dims briefly at the 50% mark. Not a strobe — barely perceptible. Used only for "live" indicators. The blinking should feel like a heartbeat, not an alarm.

TRANSMITTING
replay
duration: 3s  ·  keyframes: 0% 45% 55% 100%
opacity range: 1 → 0.15  ·  easing: ease-in-out
Pattern 03
Ticker / Marquee

Continuous horizontal scroll for live status data. Constant speed, no ease-in/out. The pace should feel like a teletype — deliberate and unhurried at ~60–80px per second.

●Job queue: nominal ●99.97% uptime ●Scheduler: active ●0 missed runs this month ●Job queue: nominal ●99.97% uptime ●Scheduler: active ●0 missed runs this month

animation: translateX(-50%)
duration: linear · infinite  ·  speed: 8–12s
Pattern 04
Staggered Reveal

Page-load entrance for content. Each element fades up with a 120ms offset from the previous. One coordinated breath, not scattered twitches.

syncUsageMetrics — running
dailyDigest — scheduled
weeklyReport — idle
cleanupJob — idle
tap to replay
duration: 500ms  ·  offset: 120ms per item
easing: cubic-bezier(0.25, 0.46, 0.45, 0.94)
Pattern 05
Button Lift

On hover, primary buttons rise 2px and shift to rust. No bounce, no scale. Just a clean lift. The interaction should feel like pressing a physical key, not a toy.

hover the button
translateY: 0 → -2px  ·  duration: 200ms
bg: charcoal → rust  ·  easing: ease-out
Pattern 06
Progress / Load

An indeterminate-feeling progress bar that moves in non-linear jumps. Mimics real work being done — fast at first, slowing as it approaches completion.

Deploying jobs…
replay
duration: 3.5s  ·  easing: cubic-bezier(0.4, 0, 0.2, 1)
keyframes: 0% 15% 40% 70% 90% 100%
Easing reference
Standard
cubic-bezier(0.25, 0.46, 0.45, 0.94)

Default for all UI transitions: buttons, panels, modals. Comfortable, not sluggish.

Enter
cubic-bezier(0.0, 0.0, 0.2, 1.0)

Elements entering the screen: drawers, toasts, page content. Decelerates into place.

Exit
cubic-bezier(0.4, 0.0, 1.0, 1.0)

Elements leaving the screen. Accelerates out — gone quickly, no lingering.

Linear
linear

Tickers, looping progress indicators, rotation. Anything that should feel mechanical and constant.

Motion anti-patterns
Never

Spring/bounce physics. The tower doesn't bounce. Nothing bounces.

Never

Scale transforms on buttons (>1.02x). Lifts only, never swells.

Never

Fast micro-animations under 150ms. Signal doesn't rush.

Never

Decorative looping animations in the product UI. Reserve motion for meaning.

04 — Voice & Tone

Transmitting,
not selling.

Signal speaks like infrastructure. Confident because it's reliable, not because it's loud.

Quiet confidence
States what's true.
Doesn't explain why that's impressive.
DO "Set it up. It runs."
NO "Our best-in-class scheduler guarantees 99.9% uptime with enterprise-grade reliability."
DO "0 missed executions this month."
NO "We're proud to announce zero missed job executions in the past 30 days!"
Utilitarian precision
Exact words. No filler. No softening qualifiers.
DO "Job failed. Retry in 30s."
NO "We noticed something may have gone wrong. Don't worry, we're on it!"
DO "3 jobs running. 847ms avg."
NO "Your jobs are processing right now at a great average speed!"
Indifferent to weather
Error messages that inform, not alarm.
DO "Timeout after 30s. Check your handler's external dependencies."
NO "Uh oh! Something went wrong 😬 Please try again or contact support."
DO "Rate limited. Backoff: 2min."
NO "Oops! It looks like we hit a rate limit. Hang tight while we sort this out!"
Patina, not polish
Documentation that assumes intelligence.
DO "Handler receives the job context. Throw to trigger retry."
NO "Welcome! In this step, we'll show you how your handler function works — exciting stuff!"
DO "Idempotency keys prevent duplicate runs. Generate once per dispatch."
NO "Great news! Signal's idempotency key feature makes sure your jobs won't run twice."
05 — Iconography

Linework.
No fills.

Icons are engineering diagrams, not illustrations. 1.5px stroke, consistent cap/join, no decoration.

Stroke rules

Always 1.5px stroke-width. Round linecap, round linejoin. No fill except the signal-green dot. Viewbox: 28×28.

Color rules

Rust (#8B5A2B) on light backgrounds. Concrete (#F4F1EC) on dark. Never patina green for icons — that's reserved for status only.

Never

Filled icons. Rounded-corner icon containers/badges. Emoji. Icons from icon packs without adapting to 1.5px stroke weight.

Layout grid

Max content width: 1100px. Page padding: 3rem (desktop) / 1.25rem (mobile). Internal grids use CSS grid with gap: 1.5px and a rust-tinted background to create the bordered-grid aesthetic.

The "gap as border" pattern comes from the lattice itself — negative space between structural members, not decorative lines drawn on top.

Spacing principles
DO Use 3rem+ padding on section containers. The brand needs to breathe.
DO Prefer asymmetric layouts — content weight on one side, negative space on the other.
NO Equal padding on all sides of every element. That's furniture catalog, not infrastructure.
NO Tight line-height under 1.6 for body text. Cramped text undermines the brand's patience.
